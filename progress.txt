PROGRESS — Kitchen Inventory Dashboard
=======================================

Current state (2026-05): production-ready bilingual PWA on Vercel.

SHIPPED
- Bilingual i18n: Arabic (default, RTL, Cairo) + English (/en, LTR, Inter),
  next-intl v4, ICU plurals throughout
- Supabase backend: auth (email + Google OAuth), per-user RLS, realtime
  sync for inventory_items AND storage_locations, image upload (Storage),
  activity logs; dev/offline fallback when env vars absent
- Inventory: CRUD with optimistic updates + rollback, filters (incl.
  virtual needsAttention), URL deep-linking (?status=), grid/list views
- Dynamic storage management: CRUD, capacity/occupancy, color, notes,
  item move/bulk-move, search + type filters, delete safety
- Shopping Mode: pre-session list (auto low-stock + manual), live session
  (states pending/partial/purchased/skipped, quick actions), review screen
  applying quantities to inventory, local history (20 sessions)
- Clickable dashboard KPI cards with deep-linked filters + honest
  "+N this week" trend on Total (from createdAt)
- Production hardening: error boundaries, loading.tsx everywhere, Zod
  validation server-side, Sentry (activates via NEXT_PUBLIC_SENTRY_DSN),
  structured logger, offline indicator + reduced-motion support,
  Vitest (40 tests) + Playwright config + GitHub Actions CI
